## 一、流水线顺序与数据流

```
源代码目录 (root_path: Path)
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│ Step 1  detect_corpus(root_path) -> dict                │  复用层
│ Step 2  extract_ast(code_files) -> dict                 │
│ Step 3  build_graph(extraction) -> nx.Graph             │
└─────────────────────────────────────────────────────────┘
    │
    │ graph.json (nx.Graph)
    ▼
┌─────────────────────────────────────────────────────────┐
│ Step 4  business_cluster(G, root_path, ...) -> list[Domain]  │  聚类
└─────────────────────────────────────────────────────────┘
    │
    │ domains: list[Domain]
    ▼
    ┌───────────────────────────────────┐
    │ Step 5  build_api_map(...)         │  API 映射（可选）
    │         -> list[ApiMatch]          │
    └───────────────┬───────────────────┘
                    │ api_matches: list[ApiMatch]
                    ▼
    ┌───────────────────────────────────┐
    │ Step 6  build_field_map(...)       │  字段映射（可选）
    │         -> dict                    │
    └───────────────┬───────────────────┘
                    │ field_map: dict
                    ▼
┌─────────────────────────────────────────────────────────┐
│ Step 7  label_domains(domains, ...) -> list[Domain]     │  LLM 标注
└─────────────────────────────────────────────────────────┘
    │
    │ domains (enriched) + api_matches + field_map
    ▼
    ┌───────────────────┐  ┌───────────────────┐
    │ Step 8            │  │ Step 9            │
    │ export_wiki(...)  │  │ export_domain_html│
    │ -> wiki/          │  │ -> .html          │
    └───────────────────┘  └───────────────────┘
```

## 二、模块接口契约

以下为每个模块必须遵守的函数签名。Step 4 的 subagent 在设计文档中不得偏离。

### 2.1 复用层 (reuse.py)

```python
def detect_corpus(root: Path) -> dict:
    """文件检测 → {files: {code: [...], doc: [...]}, total_files, total_words}"""

def extract_ast(code_files: list[Path]) -> dict:
    """AST 提取 → {nodes: [...], edges: [...], input_tokens, output_tokens}"""

def build_graph(extraction: dict) -> nx.Graph:
    """构建 NetworkX 图（无向图）"""

def merge_graph(new_extraction: dict, graph_path: str,
                prune_sources: list[str] | None = None) -> nx.Graph:
    """增量合并已变更文件的 AST 到现有图"""

def build_manifest(files: list[Path], domains: list[Domain]) -> dict:
    """生成文件哈希清单，供增量更新检测变更
    返回格式见架构文档 §6.4"""
```

### 2.2 聚类模块 (cluster.py)

```python
def business_cluster(
    G: nx.Graph,
    root_path: Path,
    language: Language = Language.AUTO,
    merge_threshold: float = 0.3,
    min_domain_size: int = 5,
) -> list[Domain]:
    """
    五步法：
    1. filter_noise(G) -> list[dict]            # 过滤噪音
    2. extract_anchors(business_nodes) -> list[dict]  # 提取锚点
    3. cluster_by_package(anchors, ...) -> dict  # 包路径聚类
    4. adjust_by_imports(candidates, G, ...) -> list[Domain]  # import 调整
    5. extract_business_points(domains, ...) -> list[Domain]  # 业务点提取
    """
```


## 三、共享数据模型

所有数据类定义在 `models.py`，类型定义以本章为准。

```python
# ── 枚举 ──

class Language(Enum):
    JAVA = "java"
    JAVASCRIPT = "javascript"
    TYPESCRIPT = "typescript"
    PYTHON = "python"
    GO = "go"
    AUTO = "auto"

class NodeRole(Enum):
    CONTROLLER = "controller"
    SERVICE_IMPL = "service_impl"
    SERVICE_API = "service_api"
    MAPPER = "mapper"
    DAO = "dao"
    ADAPTER = "adapter"
    ENTITY = "entity"
    DTO = "dto"
    VO = "vo"
    ENUM = "enum"
    HANDLER = "handler"
    CONFIG = "config"
    UTIL = "util"
    NOISE = "noise"

ANCHOR_ROLES = {NodeRole.CONTROLLER, NodeRole.SERVICE_IMPL, NodeRole.SERVICE_API, NodeRole.MAPPER, NodeRole.DAO, NodeRole.ADAPTER}

# ── 聚类模块数据类 ──

@dataclass
class BusinessPoint:
    name: str                                    # 方法名
    display_name: str = ""                       # LLM 标注的中文名
    entry_method: str = ""                       # 入口方法节点 ID
    entry_file: str = ""                         # 入口方法所在文件
    call_chain: list[str] = field(default_factory=list)
    cross_domain_calls: dict[str, list[str]] = field(default_factory=dict)
    internal_calls: list[str] = field(default_factory=list)
    infrastructure_calls: list[str] = field(default_factory=list)

@dataclass
class Domain:
    id: str                                      # 英文 ID（如 "bin-data"）
    name: str = ""                               # LLM 标注的中文名
    packages: list[str] = field(default_factory=list)
    modules: list[str] = field(default_factory=list)
    frontend_views: list[str] = field(default_factory=list)
    anchors: dict[str, list[dict]] = field(default_factory=dict)  # role → anchor_nodes
    business_points: list[BusinessPoint] = field(default_factory=list)
    total_files: int = 0
    dependencies: dict[str, int] = field(default_factory=dict)

    def anchors_flat(self) -> list[dict]:
        result = []
        for role_nodes in self.anchors.values():
            result.extend(role_nodes)
        return result

    def anchors_count(self) -> int:
        return len(self.anchors_flat())

# ── API 映射数据类 ──

@dataclass
class FrontendApiCall:
    function_name: str
    http_method: str
    url: str
    params: list[dict] = field(default_factory=list)
    source_file: str = ""
    source_line: int = 0
    callers: list[dict] = field(default_factory=list)

@dataclass
class BackendEndpoint:
    controller_file: str = ""
    controller_class: str = ""
    method_name: str = ""
    http_method: str = ""
    url: str = ""
    param_type: Optional[str] = None
    param_fields: list[dict] = field(default_factory=list)
    service_chain: list[str] = field(default_factory=list)
    return_type: Optional[str] = None

@dataclass
class ApiMatch:
    id: str
    frontend: FrontendApiCall = field(default_factory=FrontendApiCall)
    backend: BackendEndpoint = field(default_factory=BackendEndpoint)
    match_confidence: float = 0.0
    domain: str = ""
```

## 四、设计约定

所有模块 subagent 在编写设计文档时必须遵守以下约定：

### 4.1 命名规范

- **模块文件**：`{功能名词}.py`（如 `cluster.py`、`label.py`）
- **核心函数**：`{动词}_{对象}()`（如 `build_api_map`、`label_domains`、`export_wiki`）
- **内部函数**：`_{动词}_{对象}()` 前缀下划线（如 `_extract_endpoints`）

### 4.2 类型标注

所有函数签名必须包含完整类型标注（参数类型 + 返回值类型）。

### 4.3 异常处理

每个模块必须定义自己的异常基类：
```python
class ModuleNameError(Exception): pass
```
pipeline 层的 `cmd_build()` 负责捕获各模块异常并做优雅降级。

降级原则：
- `cluster.py` 失败 → 终止构建（核心链路）
- `api_mapper.py` / `field_mapper.py` 失败 → 跳过，api-docs/data-flow 为空
- `label.py` 失败 → 域保持未标注状态，使用 `domain.id` 作为显示名
- `export.py` / `visualize.py` 失败 → 跳过对应输出

### 4.4 Token 成本

- `cluster.py` / `api_mapper.py` / `field_mapper.py` / `export.py` / `visualize.py` → **零 Token**（纯静态解析）
- `label.py` → **唯一消耗 Token 的模块**（~70,000 tokens 输入 + 7,500 tokens 输出）

### 4.5 文档模板

每个模块设计文档必须包含以下章节：
1. 模块职责
2. 接口定义（完整函数签名 + 参数说明）
3. 算法步骤（伪代码或流程图）
4. 数据模型（本模块使用的类型）
5. 依赖关系（上游模块 + 外部库）
6. 错误处理
7. 测试用例（至少 3 个）

## 五、Subagent 编写顺序

按数据依赖的拓扑顺序分配：

| 顺序 | 模块 | 依赖 | 预计行数 |
|:--:|------|------|:--:|
| 1 | models.py + reuse.py | 无（底层） | 共享数据模型 |
| 2 | cluster.py | reuse, models | 核心算法 |
| 3 | api_mapper.py | cluster, models | 前后端映射 |
| 4 | field_mapper.py | api_mapper, models | 字段追踪 |
| 5 | label.py | cluster, models | LLM 标注 |
| 6 | export.py | 全部上游 | Wiki 导出 |
| 7 | visualize.py | cluster | D3 可视化 |
| 8 | pipeline.py | 全部模块 | CLI 编排 |
