# 2026-06-29 项目现状评估与完善记录

## 评估范围

- 代码仓库：Graph-Wiki
- 真相源：`docs/architecture/graph-wiki 架构设计.md`
- 配套设计：`docs/design/` 下模块设计文档
- 验证方式：Python 包安装、全量 pytest、CLI 构建 smoke、Workbench 构建

## 当前结论

Graph-Wiki 已具备可运行的核心闭环：从示例代码库检测、AST 提取、业务域聚类、API 映射、Wiki 导出，到 Workbench 数据包生成。项目当前更接近“可演示/可验收的 v0.1”，而不是只停留在设计阶段。

## 已发现问题

1. **Python 包安装失败**：`pyproject.toml` 未显式限定 setuptools 包发现范围，导致顶层 `workbench/` 被误判为 Python 包，`pip install -e .[dev]` 失败。
2. **测试存在 pytest 警告**：`tests/test_svn_pipeline.py::test_svn_full_pipeline` 返回 `True`，pytest 建议测试函数只使用断言表达成功。
3. **协作文档路径不一致**：项目 `AGENTS.md` 中仍引用旧路径 `docs/architecture/graph-wiki-design.md`，实际真相源文件为 `docs/architecture/graph-wiki 架构设计.md`。
4. **前端验证依赖本地安装**：Workbench 构建需要先执行 `npm ci` 或 `npm install` 生成 `node_modules/`。

## 本轮完善

- 在 `pyproject.toml` 增加 `tool.setuptools.packages.find`，只发布 `graph_wiki*` 包，并排除 `workbench*`、`tests*`、`docs*`。
- 删除 `test_svn_full_pipeline` 的无意义返回值，消除 pytest 返回值警告。
- 在 `.gitignore` 增加 `.venv/`，避免本地验证虚拟环境进入版本控制。
- 修正 `AGENTS.md` 中架构真相源文件路径。

## 后续建议

1. 将 `build-report.json` 的顶层质量状态与 Phase 3/4 验收状态关系再明确：当前 smoke 构建顶层 `quality.status=passed`，但 Phase 3/4 acceptance 可为 `failed`，容易让使用者误解。
2. 为前端 Workbench 增加 README 中的“一键验证”命令，例如 `npm ci && npm run build`。
3. 继续补强 frontend-only 项目的 Phase 4 覆盖解释，避免字段流为空时影响分析验收信息显得像失败而不是“不适用/样例项目限制”。
